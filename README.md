# 🎉 نظام إدارة الكاشير - إصدار v12

## 🆕 ما الجديد في هذا الإصدار

### 1️⃣ زر الرجوع في الهاتف ⬅️
- الضغط على زر "رجوع" الفيزيائي في الهاتف يعود للصفحة السابقة داخل التطبيق بدلاً من الخروج
- يُغلق أولاً أي نافذة حوار/معالج تقفيلة مفتوحة قبل التنقل
- يدعم **الروابط المباشرة** (`#closing`, `#safe`, etc.) — يمكن مشاركة رابط صفحة معينة

### 2️⃣ صفحة المصاريف العادية — أزرار جديدة
لكل مصروف الآن 4 أزرار بدل 2:
- 👁 **عرض التفاصيل** — جدول كامل بكل المعلومات
- 🔄 **تحويل النوع** (برتقالي) — يفتح حواراً لاختيار: قسم / عامة / إدارة
- ✏️ **تعديل** (السابق)
- 🗑 **حذف** (السابق)

**مهم:** التحويل يُغيّر النوع فقط دون تعديل التقفيلة المرتبطة (حسب طلبك).

### 3️⃣ صفحة المصاريف العامة — إدخال مباشر
زران جديدان بارزان أعلى الصفحة:
- 🟠 **إضافة مصروف** — نموذج كامل (وصف + مبلغ + نوع عامة/إدارة + تاريخ + قسم مرتبط اختياري)
- 🟡 **إضافة سحب** — يحفظ في 3 أماكن تلقائياً:
  - سجل السحوبات
  - سجل الديون (للتتبع)
  - سجل المصاريف العامة (لحساب الإجمالي)

### 4️⃣ صفحة الخزنة — عرض التفاصيل
زر 👁 جديد بجانب كل معاملة يُظهر:
- النوع (إيداع/سحب) + المبلغ بالألوان
- الملاحظة والتاريخ وبواسطة مَن
- إن كانت مرتبطة بتقفيلة: تفاصيل التقفيلة + زر "عرض التقفيلة" المباشر
- شارة 🔗 "مرتبطة بتقفيلة" تظهر في قائمة المعاملات

### 5️⃣ كشف الحسابات — تبويبان 🗂️
الصفحة الآن فيها **تبويبان في الأعلى**:

#### 🔵 التبويب 1: صافي التقفيلات (الناتج النهائي)
كما كان سابقاً — جمع الصوافي النهائية بعد خصم كل شيء.

#### 🟠 التبويب 2: رصيد الكاشير الخام
**هذا ما طلبته بالضبط:**
- **القسم الأول:** مبيعات كل يوم عمودياً (رصيد الكاشير قبل أي خصم) مع مجموعها
- **القسم الثاني:** الخصومات تفصيلياً (مرتجعات، مصاريف قسم، غداء، ديون، سحوبات) مع المجموع
- **القسم الثالث:** المعادلة النهائية بشكل صريح: `المبيعات − الخصومات = الناتج`

كل تبويب يعمل بالنمطين (عمودي + أفقي).

## 🧪 طريقة الاختبار

### اختبار زر الرجوع
1. افتح التطبيق
2. اضغط على "التقفيلة"
3. اضغط زر الرجوع الفيزيائي في الهاتف → يعود للواجهة الرئيسية ✅
4. إذا كان هناك modal مفتوح → يُغلقه أولاً

### اختبار تحويل المصاريف
1. اذهب لصفحة "المصاريف"
2. اختر مصروف واضغط 🔄 البرتقالي
3. اختر "عامة" أو "إدارة"
4. افتح صفحة "المصاريف العامة" → ستجد المصروف هناك

### اختبار إضافة سحب
1. "المصاريف العامة" → 🟡 "إضافة سحب"
2. اختر/أدخل اسم شخص، مبلغ، ملاحظة، نوع (عامة/إدارة)
3. احفظ → سيظهر في 3 أماكن: المصاريف العامة / الديون / السحوبات

### اختبار كشف الحسابات
1. "كشف الحسابات" → اختر شهر
2. بدّل بين التبويبين (أزرق/برتقالي)
3. بدّل بين "عمودي/أفقي" — 4 تركيبات ممكنة
4. تحقق أن: `رصيد خام − خصومات = صافي` في كلا التبويبين

## 🔧 ملاحظات تقنية

### بنية المصاريف
```javascript
// expenseEntries item:
{
  id: "...",
  amount: 50000,
  desc: "إيجار",
  type: "dept" | "general" | "admin",
  cashier: "كاشير النساء" | "",
  date: "2026-04-17",
  by: "كرار يوسف",
  fromWithdraw: true        // إن كان منشأ من "إضافة سحب"
}
```

### بنية السحوبات العامة
```javascript
// withdrawals item (جديد):
{
  id: "...",
  person: "علي",
  amount: 30000,
  note: "[عامة] فواتير",
  generalType: "general" | "admin",   // جديد
  cashier: "",
  date: "...",
  by: "..."
}
```

### history API
- `navigate()` تدفع state جديد لكل تنقل
- `popstate` listener يتعامل مع زر الرجوع
- يقرأ `location.hash` عند التحميل للسماح بالروابط المباشرة (مثل `/cashier/index.html#safe`)

## ⚠️ تنبيه

بعد الرفع قد تحتاج **قفل وإعادة فتح التطبيق** (في الموبايل) لالتقاط `sw.js` الجديد. إن لم تظهر الأزرار الجديدة:
1. افتح التطبيق في متصفح عادي (Chrome)
2. DevTools → Application → Service Workers → Unregister
3. أعد التحميل بـ Ctrl+Shift+R
4. أعد تثبيت PWA
